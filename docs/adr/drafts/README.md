# Draft ADRs

Draft ADRs contain proposals, alternatives, tradeoff discussions, and unresolved questions.

Draft ADRs are not repository policy. Only accepted, numbered ADRs under `docs/adr/` should be considered authoritative.

Once a draft decision is explicitly approved, summarize the final decision in the next numbered ADR under `docs/adr/`.

Historical discussion may remain in the draft or be archived after the approved ADR is created.

Do not create draft ADRs for topics that have not yet been discussed.
